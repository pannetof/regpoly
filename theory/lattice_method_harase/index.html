    Fast Lattice Reduction for Equidistribution Testing¶

    Harase, Matsumoto, and Saito (2011)¶

    References¶

    • Harase, S., Matsumoto, M., and Saito, M. (2011). "Fast Lattice Reduction for \(\mathbb{F}_2\)-Linear Pseudorandom Number Generators." Mathematics of Computation, 80(273):395--407.
    • Harase, S. (2011). "An Efficient Lattice Reduction Method for \(\mathbb{F}_2\)-Linear Pseudorandom Number Generators Using Mulders and Storjohann Algorithm." J. Comput. Appl. Math., 236(2):141--149.
    • Mulders, T. and Storjohann, A. (2003). "On lattice reduction for polynomial matrices." J. Symbolic Computation, 35:377--401.

    1. Motivation¶

    The Couture-L'Ecuyer method (2000) computes \(t_\ell\) for each resolution \(\ell\) independently by reducing the dual lattice \(L_\ell'\) with Lenstra's algorithm. The Harase-Matsumoto-Saito method improves this in three ways:

    1. Works with the primal lattice instead of the dual, avoiding the matrix inversion needed to construct the dual.
    2. Represents basis vectors in the state space \(\mathbb{F}_2^k\) rather than as explicit polynomials, making the shift operation \(z \cdot a(z) \bmod P(z)\) equivalent to applying the transition matrix \(A\) -- which is \(O(1)\) for sparse generators (Tausworthe, MT, WELL).
    3. Replaces Lenstra's algorithm with the Mulders-Storjohann (MS) weak reduction, which has a simpler inner loop and fewer branches.

    2. Problem Statement¶

    Same as Couture-L'Ecuyer: compute \(\delta_\ell = \lfloor k/\ell \rfloor - t_\ell\) for \(\ell = 1, \ldots, L\), where \(t_\ell\) is the dimension of equidistribution with \(\ell\)-bit accuracy.

    3. Primal Lattice¶

    Definition¶

    For resolution \(\ell\), the primal lattice \(L_\ell\) is the \(\mathbb{F}_2[z]\)-module of rank \(\ell\) in \(\mathbb{K}^\ell\) (where \(\mathbb{K} = \mathbb{F}_2((z^{-1}))\)) generated by the columns of:

    \[\begin{pmatrix} 1/P(z) & 0 & \cdots & 0 \\ h_1(z)/P(z) & 1 & \cdots & 0 \\ \vdots & & \ddots & \vdots \\ h_{\ell-1}(z)/P(z) & 0 & \cdots & 1 \end{pmatrix}\]

    Equivalently, using the primal basis:

    \[B_\ell = \begin{pmatrix} 1 & h_1 & h_2 & \cdots & h_{\ell-1} \\ 0 & P & 0 & \cdots & 0 \\ 0 & 0 & P & \cdots & 0 \\ \vdots & & & \ddots & \vdots \\ 0 & 0 & 0 & \cdots & P \end{pmatrix}\]

    where the first row has degree \(< k\) entries and the remaining rows have \(P(z)\) on the diagonal.

    Relationship to equidistribution¶

    By the duality theorem (Couture-L'Ecuyer 2000, Theorem 1), \(t_\ell\) equals the first successive minimum \(\lambda_1\) of either \(L_\ell\) or \(L_\ell'\). The Harase method computes \(\lambda_1(L_\ell)\) via reduction of the primal basis.

    4. State-Space Representation¶

    Key idea¶

    A polynomial \(a(z) \bmod P(z)\) of degree \(< k\) can be represented as a vector in \(\mathbb{F}_2^k\) (its coefficient vector). All polynomial operations modulo \(P(z)\) become matrix operations:

    Polynomial operation State-space operation
    \(a(z) + b(z) \bmod P\) \(\mathbf{a} \oplus \mathbf{b}\) (bitwise XOR)
    \(z \cdot a(z) \bmod P\) \(A \cdot \mathbf{a}\) (apply transition matrix)
    \(z^s \cdot a(z) \bmod P\) \(A^s \cdot \mathbf{a}\)

    For structured generators (MT, WELL, Tausworthe, MELG), applying \(A\) is the same as one step of the generator: \(O(N)\) word operations where \(N = k/w\). This replaces the \(O(k/64)\) cost of a polynomial shift.

    Basis vector as state vectors¶

    Each row of the primal basis is an \(\ell\)-tuple of polynomials mod \(P(z)\). In state-space form, it becomes an \(\ell\)-tuple of \(k\)-bit vectors:

    \[\mathbf{b}_i = (\mathbf{s}_{i,0},\; \mathbf{s}_{i,1},\; \ldots,\; \mathbf{s}_{i,\ell-1}) \qquad \text{where each } \mathbf{s}_{i,j} \in \mathbb{F}_2^k\]

    The norm \(\|\mathbf{b}_i\|\) is the maximum degree across all coordinates:

    \[\|\mathbf{b}_i\| = \max_j \deg(\text{polynomial represented by } \mathbf{s}_{i,j})\]

    Computing the degree of a \(k\)-bit vector = finding the position of the highest set bit, which is \(O(k/64)\).

    5. Mulders-Storjohann Weak Reduction¶

    The MS algorithm is a simpler alternative to Lenstra's algorithm for computing successive minima of polynomial lattices.

    Algorithm¶

    Input: \(\ell \times \ell\) basis \(\{\mathbf{b}_0, \ldots, \mathbf{b}_{\ell-1}\}\) with \(\|\mathbf{b}_0\| \le \|\mathbf{b}_1\| \le \cdots \le \|\mathbf{b}_{\ell-1}\|\) (sorted).

    Output: Weakly reduced basis, with \(\|\mathbf{b}_0\| = \lambda_1(L_\ell)\).

    MS-REDUCE(basis, ell):
    // Sort rows by non-decreasing norm
    sort basis[0..ell-1] by ||b_i|| ascending

    changed := true
    WHILE changed:
        changed := false
        FOR i = 0 TO ell - 2:
            FOR j = i + 1 TO ell - 1:
                // Check if b_i can reduce b_j
                s := ||b_j|| - ||b_i||
                IF s >= 0:
                    // Check if the leading coefficient of z^s * b_i
                    // matches the leading coefficient of b_j in some coordinate
                    IF leading_term_cancels(b_j, b_i, s):
                        b_j := b_j + z^s * b_i     // XOR with shifted row
                        update ||b_j||
                        IF ||b_j|| < ||b_i||:
                            swap b_i and b_j
                        re-sort rows i..ell-1 by norm
                        changed := true
                        BREAK inner loop (restart from i)

    RETURN ||b_0||

    Leading term cancellation check¶

    The reduction \(\mathbf{b}_j \leftarrow \mathbf{b}_j + z^s \cdot \mathbf{b}_i\) cancels the leading term of \(\mathbf{b}_j\) if: there exists a coordinate \(c\) such that the coefficient of \(z^{\|\mathbf{b}_j\|}\) in coordinate \(c\) is 1 in both \(\mathbf{b}_j\) and \(z^s \cdot \mathbf{b}_i\).

    Since \(\|\mathbf{b}_j\| = \|\mathbf{b}_i\| + s\), this means: the leading coefficient of \(\mathbf{b}_i\) (at degree \(\|\mathbf{b}_i\|\)) in coordinate \(c\) is 1, and the coefficient of \(\mathbf{b}_j\) at degree \(\|\mathbf{b}_j\|\) in the same coordinate \(c\) is also 1.

    In practice: check if indicemaxdeg of \(\mathbf{b}_i\) matches a coordinate where \(\mathbf{b}_j\) also has its leading bit set.

    Shift in state-space¶

    The operation \(z^s \cdot \mathbf{b}_i\) shifts all coordinate polynomials by \(s\). In state-space representation:

    \[z^s \cdot \mathbf{s}_{i,c} = A^s \cdot \mathbf{s}_{i,c}\]

    For large \(s\), this is computed by repeated squaring: express \(s\) in binary and apply the appropriate powers of \(A\). But if \(A\) is sparse (e.g., Tausworthe with trinomial), applying \(A\) once is \(O(1)\) word operations, and applying \(A^s\) is \(O(s \cdot N/64)\) where \(N/64\) is the number of words per state.

    For practical implementations, \(s \le k\) and the shift is applied by running the generator \(s\) steps. This costs \(O(s \cdot N / 64)\) word operations per coordinate, or \(O(s \cdot \ell \cdot k / 64)\) for all coordinates.

    Complexity per resolution¶

    The MS algorithm performs at most \(O(\ell^2)\) reduction steps. Each reduction step involves one shift (\(O(k/64)\) per coordinate) and one XOR (\(O(k/64)\) per coordinate), for a total of \(O(\ell \cdot k/64)\) per step.

    Total per resolution: \(O(\ell^3 \cdot k / 64)\) word operations.

    This is comparable to Lenstra's \(O(\ell^2 \cdot k / 64)\) per resolution. The practical advantage of MS is its simpler inner loop (fewer branches, no solve_axb subsystem).

    6. Incremental Algorithm Across Resolutions¶

    Growing the basis from \(\ell\) to \(\ell + 1\)¶

    Same principle as Couture-L'Ecuyer: add one new row to the basis and re-reduce.

    1. Extend all existing rows with a zero in coordinate \(\ell\).
    2. Add new row \(\mathbf{b}_\ell = (h_\ell(z), 0, \ldots, 0, P(z))\) where \(h_\ell\) is in coordinate 0 and \(P\) is in coordinate \(\ell\).
    3. Run MS-REDUCE on the \((\ell + 1) \times (\ell + 1)\) basis.

    Since the basis was already reduced for resolution \(\ell\), most of the previous reduction work carries over. Only the new row needs to be integrated.

    Total complexity for all resolutions¶

    \[\text{Total} = \sum_{\ell=1}^{L} O(\ell^3 \cdot k / 64) = O(L^4 \cdot k / 64)\]

    In practice, the re-reduction after adding one row is much cheaper than a full reduction from scratch, so the actual cost is closer to \(O(L^3 \cdot k / 64)\).

    7. Comparison of the Two Methods¶

    Aspect Couture-L'Ecuyer (2000) Harase et al. (2011)
    Lattice Dual \(L_\ell'\) Primal \(L_\ell\)
    Reduction Lenstra Mulders-Storjohann
    Representation Explicit polynomials State-space vectors
    Shift cost \(O(k/64)\) word ops \(O(N/64)\) for sparse generators
    Per resolution \(O(\ell^2 \cdot k/64)\) \(O(\ell^3 \cdot k/64)\) theoretical, \(O(\ell^2 \cdot k/64)\) practical
    All resolutions \(O(L^3 \cdot k/64)\) \(O(L^3 \cdot k/64)\) practical
    Preprocessing \(O(L \cdot k^2/64)\) (generating polys) + GCD inversion \(O(L \cdot k^2/64)\) (generating polys)
    Extra NTL dependency Yes (for \(g_0^{-1} \bmod P\)) No (pure bit operations)
    Practical speedup baseline ~3x observed (Harase 2011, Table 1)

    When each method is preferred¶

    • Couture-L'Ecuyer: proven, well-understood, good for moderate \(k\) and \(L\). Requires NTL for the polynomial inversion step.
    • Harase: better for large \(k\) with sparse transition matrices. Avoids the NTL dependency for the lattice reduction itself (NTL may still be needed for the characteristic polynomial).

    8. Initialization of the Primal Basis (Harase variant)¶

    For resolution \(\ell = 1\)¶

    The primal lattice \(L_1\) is one-dimensional with basis \(\{(1/P(z))\}\). In state-space form, the basis vector is the state that generates the sequence \(y_0(z) = g_0(z)/P(z)\). Its degree is \(\deg(g_0) < k\).

    For resolution \(\ell\)¶

    The \(\ell \times \ell\) primal basis is:

    Row Coordinate 0 Coordinate 1 \(\cdots\) Coordinate \(\ell-1\)
    0 \(1\) (= state for \(y_0/P\)) \(h_1\) \(\cdots\) \(h_{\ell-1}\)
    1 \(0\) \(P\) \(\cdots\) \(0\)
    \(\vdots\) \(\ddots\)
    \(\ell-1\) \(0\) \(0\) \(\cdots\) \(P\)

    In state-space form: - Row 0: each coordinate \(c\) stores the \(k\)-bit state vector representing \(h_c(z)\) (the normalized generating polynomial for output bit \(c\)). Coordinate 0 stores the identity polynomial (\(= e_0\) in state space). - Rows \(1, \ldots, \ell-1\): coordinate \(c\) stores \(P(z)\) (= the zero state, since \(P(z) \equiv 0 \bmod P(z)\) in the residue ring) for \(c \ne i\), and \(P(z)\) (represented as a full polynomial, NOT reduced mod \(P\)) for \(c = i\).

    Important subtlety: in the primal basis, the diagonal entries of rows \(1, \ldots, \ell-1\) are \(P(z)\) itself, which has degree \(k\). Since \(P(z) \bmod P(z) = 0\), these cannot be stored in the residue representation. They must be stored as full \(k+1\)-bit polynomials. The degree of rows \(1, \ldots, \ell-1\) is exactly \(k\).

    9. Harase's Computation of \(h_j\) Without Inversion¶

    In the Couture-L'Ecuyer method, the normalized polynomials \(h_j = g_j \cdot g_0^{-1} \bmod P\) require computing \(g_0^{-1} \bmod P\) using NTL.

    Harase avoids this by working directly with the un-normalized generating polynomials \(g_j\) and absorbing the normalization into the lattice basis:

    1. Set row 0 of the primal basis to \((g_0,\; g_1,\; \ldots,\; g_{\ell-1})\).
    2. Set rows \(1, \ldots, \ell-1\) as before with \(P\) on the diagonal.
    3. The lattice reduction itself will handle the effective normalization (since \(g_0\) and \(P\) span the same sublattice as \(1\) and \(P\) up to a unit in the ring).

    This eliminates the NTL dependency for the lattice reduction phase.

    10. Summary: Implementation Checklist¶

    For an implementer, the steps to add the Harase method are:

    1. Compute generating polynomials \(g_0, \ldots, g_{L-1}\) by running the generator from canonical state for \(k\) steps (same as Couture-L'Ecuyer step 2).

    2. Build the initial primal basis for resolution 1:

    3. Row 0: coordinate 0 = \(g_0\) (as a \(k\)-bit vector).

    4. (For later: row 1 will have \(P\) on the diagonal.)

    5. For each resolution \(\ell = 1, \ldots, L\):

    6. If \(\ell > 1\): extend the basis (add coordinate \(\ell - 1\) to existing rows, add new row with \(P\) in coordinate \(\ell - 1\)).
    7. Run MS-REDUCE.
    8. Read \(\lambda_1 = \|\mathbf{b}_0\|\) = first successive minimum.
    9. \(t_\ell = \min(\lambda_1, \lfloor k/\ell \rfloor)\).

    10. \(\delta_\ell = \lfloor k/\ell \rfloor - t_\ell\).

    11. Data structures needed:

    12. PolVect: \(\ell\)-tuple of BitVect(\(k+1\)), plus deg and indicemaxdeg.
    13. Basis: array of \(L\) PolVects.

    14. Shift operation: either explicit polynomial shift (\(O(k/64)\)) or apply generator transition (\(O(N/64)\) for sparse generators).

    15. Key operations to implement:

    16. shift_and_xor(b_j, b_i, s): \(\mathbf{b}_j \leftarrow \mathbf{b}_j + z^s \cdot \mathbf{b}_i\)
    17. compute_norm(b): find highest set bit across all coordinates
    18. leading_term_cancels(b_j, b_i, s): check if shift-XOR reduces the norm
    19. sort_by_norm(basis, start, end): re-sort basis rows
