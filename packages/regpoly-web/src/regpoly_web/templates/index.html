regpoly

Search and analysis for pseudo-random number generators based on linear recurrences over GF(2).

What this tool does

regpoly helps you design high-quality linear pseudo-random number generators. You pick a generator family (Tausworthe, Mersenne Twister, WELL, MELG, …), search for parameter choices that give the generator its maximum possible period (full-period), combine full-period generators into multi-component XOR generators, apply a tempering transformation to improve statistical quality, and evaluate the result against formal equidistribution and collision-free criteria.

Every result is persisted in a database and can be revisited, filtered, and exported later. Long-running searches run in worker processes and can be paused, resumed, cancelled, and restarted from the Searches page.

Get started

Pick a generator family from the sidebar on the left. Each family exposes five activities — the tasks below are available either scoped to that family (via the family page) or unscoped (via the Global section of the sidebar).

Full-period generator search
Find new full-period generators in this family. You fix the structural parameters (the ones that define the state size k) and regpoly randomises the remaining parameters, checking each candidate's characteristic polynomial for primitivity. Full-period generators attain the theoretical maximum period of 2k − 1.
Full-period generators
Browse generators already found. Filter by family, by k, or by any structural or non-structural parameter. Each generator page shows its characteristic polynomial, its Hamming weight, its equidistribution profile, and its transition matrix, all computed automatically in the background as soon as it is discovered.
Combined generator search
Search for combined generators. Pick one or more components, feed each component a pool of full-period generators, choose a tempering transformation (Matsumoto– Kurita, lagged masking, permutation, …), and run a formal quality test (equidistribution, collision-free, tuplets) over every valid combination. When the transformation has optimisable bitmasks, an inner optimiser tunes them automatically.
Combined generators
Browse tested combined generators. See every generator the search produced, trace each back to the run that found it, inspect the test result (dimension defects, collision ranks, tuplets thresholds) and export the combined configuration as a YAML file.
Documentation (per family)
Recurrence equations and academic references. Every family carries its own write-up — the mathematical recurrence that defines the generator, the meaning of each parameter, and the original publications (Matsumoto, L'Ecuyer, and others). Open a family page from the sidebar and click the Documentation link.
